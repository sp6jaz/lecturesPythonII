## 1. Rubryka oceny prezentacji (0-10 pkt)

### Karta oceny prezentacji — dla prowadzącego

| Kryterium | 0 pkt | 1 pkt | 2 pkt |
|-----------|-------|-------|-------|
| **1. Struktura** — problem -> dane -> metoda -> wyniki -> wnioski | Brak struktury, chaotycznie, nie wiadomo o czym jest analiza | Częściowa struktura, brakuje elementów (np. brak wniosków lub opisu danych) | Jasna struktura, logiczny przepływ od pytania do odpowiedzi |
| **2. Dataset i pipeline** — kompletność analizy (EDA -> czyszczenie -> wizualizacja -> statystyka) | Tylko surowe dane lub 1 krok analizy | 2-3 kroki pipeline'u (np. wczytanie + wykres, ale brak czyszczenia) | Pełny pipeline (4+ kroków): EDA, czyszczenie, wizualizacja, wnioski/statystyka |
| **3. Wizualizacje** — czytelność, etykiety, tytuły, dobór typu wykresu | Brak wykresów lub nieczytelne (brak osi, za małe, nieodpowiedni typ) | Wykresy są, ale brak etykiet/tytułów lub nieodpowiedni typ wykresu | Czytelne wykresy z pełnym opisem, dobrze dobrany typ do danych |
| **4. Wnioski** — poparte danymi, relewantne biznesowo | Brak wniosków lub oderwane od danych | Wnioski ogólne, niepoparte konkretnymi liczbami | Konkretne wnioski z liczbami, odniesienie do pytania badawczego |
| **5. Rozumienie kodu** — student potrafi wyjaśnić co robi jego kod | Nie potrafi wyjaśnić podstawowych operacji | Wyjaśnia częściowo, z pomocą prowadzącego | Wyjaśnia samodzielnie i pewnie, odpowiada na pytania dodatkowe |

**Suma prezentacji: ___ / 10 pkt**

### Szybki klucz oceniania

| Punkty | Poziom | Opis |
|--------|--------|------|
| 9-10 | Wzorowy | Pełna analiza, profesjonalne wizualizacje, pewne odpowiedzi |
| 7-8 | Dobry | Kompletna analiza z drobnymi brakami, rozumie kod |
| 5-6 | Dostateczny | Podstawowa analiza, 1-2 wykresy, wnioski ogólne |
| 3-4 | Słaby | Niekompletna analiza, problemy z wyjaśnieniem kodu |
| 0-2 | Niedostateczny | Brak analizy lub kompletnie chaotyczna prezentacja |

## 2. Rubryka portfolio GitHub (0-5 pkt)

### Karta weryfikacji portfolio — dla prowadzącego

| Kryterium | 0 pkt | 1 pkt | 2 pkt |
|-----------|-------|-------|-------|
| **Repozytorium + README** | Brak repo lub puste | Repo istnieje, ale brak README | Repo + README z opisem projektu/kursu |
| **Notebooki z laboratoriów** | <3 notebooki | 3-5 notebooków | 6+ notebooków |
| **Historia commitów** | <3 commitów (wrzucone naraz) | 3-8 commitów | 9+ commitów (regularne, z opisami) |

**Suma surowa: ___ / 6 pkt (cap: max 5 pkt)**

### Checklist szybkiej weryfikacji

```
WERYFIKACJA PORTFOLIO — prowadzący przechodzi po sali lub sprawdza zdalnie:

[ ] Repozytorium istnieje i jest publiczne (lub prowadzący ma dostęp)
[ ] README.md w repozytorium (nawet krótki)
[ ] Minimum 3 notebooki z laboratoriów (S01-S09)
[ ] Notebook z mini-projektem (ten prezentowany)
[ ] Historia commitów widoczna (git log)
[ ] Commity mają sensowne opisy (nie same "update" czy "fix")
```

### Uwagi dla zaocznych
- Studenci zaoczni mieli 10 spotkań (nie 15) — mniej notebooków jest normalne
- Akceptowalne minimum: 3 notebooki z labów + 1 notebook projektu
- Pracują zawodowo — doceniaj regularność commitów bardziej niż ich liczbę
- Jeśli student ma prywatne repo — poproś o dodanie Cię jako collaborator na miejscu

## 5. Wymagania minimalne mini-projektu

Studenci powinni byli otrzymać te wymagania na S07-S08 (zapowiedź projektu):

```
MINI-PROJEKT — WYMAGANIA MINIMALNE:

1. DATASET
   - Własny dataset (nie z zajęć) — min. 100 wierszy, 5+ kolumn
   - Źródło: Kaggle, UCI ML Repository, dane.gov.pl, API publiczne, GUS
   - Zapisany w repozytorium (CSV/Excel) lub wczytywany z URL

2. NOTEBOOK (.ipynb)
   - Komórka Markdown: tytuł, opis datasetu, pytanie badawcze
   - EDA: info(), describe(), shape, analiza brakujących danych
   - Czyszczenie: obsługa NaN, konwersja typów, duplikaty (jeśli dotyczy)
   - Minimum 3 wizualizacje (różne typy wykresów)
   - Minimum 1 analiza statystyczna (korelacja, test, groupby+agg)
   - Komórka Markdown: wnioski (minimum 3 bullet points)

3. GIT
   - Notebook na GitHubie (publiczny lub z dostępem dla prowadzącego)
   - Minimum 3 commity (nie jeden commit z wszystkim naraz)
   - README.md z krótkim opisem projektu

4. PREZENTACJA
   - 5 minut: problem -> dane -> analiza -> wykresy -> wnioski
   - Prezentacja z notebooka Jupyter (nie PowerPoint)
   - Gotowość na 1-2 pytania od prowadzącego

PROJEKTY AKCEPTOWANE BEZ PREZENTACJI (obniżona ocena):
   - Notebook spełniający punkty 1-3, przesłany terminowo
   - Krótka rozmowa z prowadzącym (5 min) zamiast prezentacji publicznej
   - Max 70% punktów za komponent prezentacji
```

### Sugerowane źródła datasetów (do wyświetlenia na projektorze)

```
GDZIE SZUKAC DATASETU:

  Kaggle Datasets      -> kaggle.com/datasets

  dane.gov.pl          -> dane.gov.pl (polskie dane publiczne)
  GUS / BDL            -> bdl.stat.gov.pl
  WHO                  -> who.int/data
  World Bank           -> data.worldbank.org
  GitHub awesome-data  -> github.com/awesomedata/awesome-public-datasets

DOBRE TEMATY DLA ANALITYKOW:
  - Sprzedaż e-commerce (Kaggle: E-Commerce Shipping)
  - HR Analytics (IBM HR Attrition)
  - Superstore Sales (Tableau sample)
  - Wine Quality (UCI)
  - Dane demograficzne GUS
  - Wyniki finansowe spółek GPW
```

## 6. Procedura dla studentów bez prezentacji

### Wariant A: Student obecny, ale nieprzygotowany

1. Ustal termin oddania notebooka: **+1 tydzień od S10** (konkretna data)
2. Student przesyła notebook na GitHub i informuje mailem
3. Prowadzący weryfikuje notebook + krótka rozmowa online/mailowa (5 min)
4. Ocena: max **70% punktów** za prezentację (7/10 zamiast 10/10)
5. Portfolio GitHub oceniane normalnie (0-5 pkt)

### Wariant B: Student nieobecny na S10

1. Jeśli usprawiedliwiony — ten sam tryb co wariant A
2. Termin: **+2 tygodnie od S10** (przed zamknięciem ocen w systemie)
3. Forma: przesłanie notebooka + krótka rozmowa online (Teams/Zoom/mail)
4. Ocena: max **70% punktów** za prezentację
5. Jeśli nieusprawiedliwiony i brak kontaktu — 0 pkt za S10

### Wariant C: Student nie zrobił projektu w ogóle

1. Zaproponuj termin awaryjny: **+2 tygodnie od S10**
2. Wymagania minimalne: notebook z EDA + 3 wizualizacje + wnioski + GitHub
3. Max **50% punktów** za prezentację (5/10)
4. Bez portfolio = 0 pkt za portfolio
5. Jeśli nie odda — 0 pkt za S10 (może mieć wpływ na zaliczenie laboratorium)
